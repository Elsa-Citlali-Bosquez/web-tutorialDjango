¿Sobre qué actúa Django?


  • Páginas: Django define la estructura de las páginas web a través de vistas. Estas vistas son funciones de Python que reciben una solicitud HTTP y retornan una respuesta, típicamente una plantilla HTML renderizada.

  • Modelos: Django utiliza modelos para representar los datos de tu aplicación. Estos modelos se mapean a tablas en una base de datos (por defecto, PostgreSQL, aunque puede usar otras). Los modelos definen los campos (por ejemplo, nombre, edad, correo electrónico) que almacenarán tus datos.

  • Plantillas: Las plantillas son archivos de texto que contienen la estructura HTML de tus páginas. Django las usa para generar el HTML dinámico que será enviado al navegador.

  • URLS: Django utiliza un sistema de URLS para mapear las direcciones web a las vistas correspondientes. De esta manera, cuando un usuario ingresa una URL en su navegador, Django sabe qué vista debe ejecutar.

  • Forms: Los forms de Django simplifican la creación de formularios web. Puedes definir campos, validar datos y generar el HTML necesario para mostrar y procesar los formularios.


¿Cómo se modifican los archivos en Django?


Es necesario familiarizarse con la estructura de directorios del proyecto. La estructura del proyecto en Django incluye:

  • models.py: Aquí se definen los modelos que representan los datos de tu aplicación.

  • views.py: Contiene las vistas que procesan las solicitudes HTTP y generan las respuestas.

  • urls.py: Define las URL de tu aplicación y las relaciona con las vistas.

  • templates: Contiene las plantillas HTML para renderizar las páginas.

    • Estos archivos se editan con cualquier editor de texto o IDE que soporte Python. La mayoría de los desarrolladores de Django utilizan editores como Visual Studio Code, PyCharm o Sublime Text. Realiza cambios según sea necesario, por ejemplo si deseas agregar una nueva vista edita views.py y define una nueva función o clase.


  • Migraciones: Si modificas modelos en models.py, recuerda ejecutar las migraciones: python manage.py makemigrations
    python manage.py migrate

  • Reinicia el servidor: Si tu proyecto está en ejecución, por lo general, con python manage.py runserver se recomienda reiniciar el servidor para que los cambios tengan efecto.


¿Cómo se referencian los archivos creados desde otras aplicaciones?


Django permite crear aplicaciones reutilizables que pueden ser incluidas en diferentes proyectos. Para referenciar archivos desde otras aplicaciones, se utilizan los siguientes mecanismos:

  • Importar modelos: Puedes importar modelos de otras aplicaciones para utilizarlos en tus propias vistas y plantillas. Por ejemplo: from otra_app.models import Producto

  • Incluir URLs: Puedes incluir las URL de otras aplicaciones en tu archivo urls.py principal. Por ejemplo: from django.urls import include, path
    from otra_app import urls as otra_app_urls
    urlpatterns = [path('otra_app/', include(otra_app_urls)), ]

  • Importar vistas: para usar vistas desde otra aplicación, simplemente importa la vista en tu archivo "urls.py" o "views.py":
    from otra_app.views import MiVista
  • Plantillas: para cargar plantillas de otra aplicación, asegúrate de que la configuración de TEMPLATES en "settings.py" incluye las rutas correctas. Luego, puedes usar el nombre de la plantilla directamente:
    {% include 'otra_app/mi_template.html' %}

  • Archivos estáticos: si necesitas referenciar archivos estáticos de otra aplicación, asegúrate de que la aplicación esté incluida en INSTALLED_APPS y utiliza la etiqueta {% static %}:
    <*link rel="stylesheet" href="{% static 'otra_app/css/estilos.css' %}"*>

    • Nota: en este último comando omite los asteriscos.
